# 内容处理链路（baoyu-skills）

> 主线目标：把外部内容稳定转成“可归档、可复用、可发布”的 Markdown/HTML。

## 0. 选路（你要处理的输入是什么？）

- 网页（需要 JS 渲染/可能需要登录）→ [[05-AI技术文档/外部技能库/baoyu-skills/技能卡片/baoyu-url-to-markdown]]
- X/Twitter 推文/长文 → [[05-AI技术文档/外部技能库/baoyu-skills/技能卡片/baoyu-danger-x-to-markdown]]（先做 Consent + 首次 EXTEND 设置）

## 1. 抓取：网页 → Markdown（Chrome CDP）

使用：[[05-AI技术文档/外部技能库/baoyu-skills/技能卡片/baoyu-url-to-markdown]]

关键要点：
- 公开网页：默认 Auto 模式即可。
- 需要登录/懒加载：用 `--wait`，等页面准备好再触发抓取。

典型产出（源文档定义）：
- `url-to-markdown/<domain>/<slug>.md`

## 2. 抓取：X → Markdown（带同意声明）

使用：[[05-AI技术文档/外部技能库/baoyu-skills/技能卡片/baoyu-danger-x-to-markdown]]

关键要点：
- **先同意声明（Consent）**，否则不应继续执行转换。
- EXTEND.md 不存在时是 **BLOCKING**：必须先完成首次偏好设置（媒体是否下载、默认输出目录、保存位置）。
- 如果开启媒体下载：会把图片/视频落到本地并重写链接，便于长期归档。

典型产出（源文档定义）：
- `x-to-markdown/{username}/{tweet-id}/{content-slug}.md`（并可能包含 `imgs/`、`videos/`）

## 3. 清洗排版：Markdown → 更可读/更规范的 Markdown

使用：[[05-AI技术文档/外部技能库/baoyu-skills/技能卡片/baoyu-format-markdown]]

建议做法：
- 默认策略：生成 `*-formatted.md`（不破坏原始抓取稿）。
- 如果只想做排版“纠错”（中英混排空格、引号等）：选择 “Typography only” 方案（注意它可能会原地修改）。

## 4. 可选：Markdown → HTML（公众号友好）

使用：[[05-AI技术文档/外部技能库/baoyu-skills/技能卡片/baoyu-markdown-to-html]]

关键要点：
- 输出默认与原 md 同目录同名 `.html`，若已存在会先备份（`.bak-YYYYMMDDHHMMSS`）。
- 主题（theme）可通过该 skill 的 EXTEND 默认化，减少每次选择。

## 5. 可选：发布

- 公众号：[[05-AI技术文档/外部技能库/baoyu-skills/技能卡片/baoyu-post-to-wechat]]
- X：[[05-AI技术文档/外部技能库/baoyu-skills/技能卡片/baoyu-post-to-x]]

> 发布链路涉及浏览器自动化/API 权限，建议先把“内容处理”链路跑顺，再接入发布。

---

## 推荐的最小闭环（可复制）

1. `baoyu-url-to-markdown` 抓取网页 → 得到 `.md`
2. `baoyu-format-markdown` 清洗排版 → 得到 `*-formatted.md`
3. （可选）`baoyu-markdown-to-html` 转出 `.html`
4. （可选）`baoyu-post-to-wechat` 发布
