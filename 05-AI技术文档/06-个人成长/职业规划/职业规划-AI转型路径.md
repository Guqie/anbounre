## 二、方向选择（基于2026年市场调研）

### 为什么不选NLP算法工程师？

市场现实很残酷：
- NLP算法岗 JD普遍要求：硕士+、顶会论文、3年+经验
- 校招月薪中位数24,940元——但那是985硕士的价格
- 你的学历（专升本）会在简历筛选阶段被直接过滤

### 为什么选大模型应用开发？

这是你当前最优解：
- JD要求与你高度匹配：LangChain、RAG、Python、Agent——你全有
- 本科可投，看重实战能力而非学历
- 你有真实的生产环境落地经验（不是课程作业）
- 社招本科1-3年薪资范围：15-25K，符合你的目标

### 方向对比

| 方向 | 匹配度 | 薪资范围 | 学历门槛 | 你的胜算 |
|------|--------|---------|---------|---------|
| 大模型应用开发 | ⭐⭐⭐⭐ | 15-25K | 本科可投 | 高——核心技能全覆盖 |
| AI应用工程师 | ⭐⭐⭐⭐⭐ | 10-18K | 本科友好 | 很高——保底选项 |
| NLP算法工程师 | ⭐⭐ | 20-35K | 硕士+论文 | 低——学历硬伤 |

### 定位

你不是"转行到AI"，你是**AI科班回归技术岗**：
- AI本科学历 + 优秀毕业论文
- 知识图谱、NLP、深度学习的完整项目经验
- 在职期间主导RAG+Agent系统开发
- 只是毕业后暂时进了研究岗，技术底子一直在

### 推荐路线

```
当前：助理研究员（7K）
  ↓ 1-2个月准备
目标一：大模型应用开发工程师（15-20K）
  ↓ 1-2年积累
目标二：高级AI工程师 / 技术负责人（25-35K）
```

## 三、执行计划

### 阶段一：求职准备期（1-2个月）

> 与 [[AI技能学习路线]] 联动

**核心原则：不是"学新东西"，而是"把已有能力打磨到面试水平" + 补齐两个硬伤**

**🔴 P0 — 必须完成（决定能不能拿到面试）：**

- [ ] 整理GitHub Portfolio，上传2个核心项目代码（KG+LLM QA系统、RAG+Agent简报系统）
- [ ] 补Docker基础：能写Dockerfile、docker-compose编排项目
- [ ] 补FastAPI：把毕业设计封装为REST API
- [ ] 复习两个核心项目，能画架构图、讲清每个技术选型的why

**🟡 P1 — 重要（决定面试能不能过）：**

- [ ] LeetCode刷题（每天1-2题，3周50题，重点：数组、链表、树、动态规划）
- [ ] 八股文复习：Transformer、BERT、RAG全流程、向量数据库原理
- [ ] 技术社区发文1-2篇（掘金/CSDN），展示技术深度

### 阶段二：求职冲刺期（第2-3个月）

- [ ] 针对目标岗位JD定制简历（参见 [[简历-AI方向]]）
- [ ] 模拟面试：项目深度讲解 + 算法手撕 + 系统设计画图
- [ ] 开始投递，每周至少投递10个岗位

**投递策略（按优先级）：**

| 优先级 | 公司类型 | 典型岗位 | 说明 |
|--------|---------|---------|------|
| ⭐⭐⭐ | 大模型应用层公司 | RAG/Agent方向开发 | 最匹配，主力投递 |
| ⭐⭐⭐ | AI+垂直行业公司 | 政务AI、产业AI、教育AI | 你有央企服务经验，差异化优势 |
| ⭐⭐ | 互联网公司AI部门 | 搜索/推荐/对话系统 | 竞争激烈但薪资高 |
| ⭐⭐ | 央企/国企数字化转型 | AI应用开发 | 稳定，但薪资可能偏低 |
| ⭐ | 传统软件公司AI组 | AI功能集成 | 保底选项 |

### 阶段三：入职成长期（入职后6-12个月）

- [ ] 快速熟悉公司AI技术栈，主动承担核心模块开发
- [ ] 积累2-3个完整AI项目从需求到上线的交付经验
- [ ] 深入一个技术方向建立壁垒（推荐：RAG优化 或 Agent架构）
- [ ] 补齐生产级工程能力：CI/CD、监控、日志、性能调优

### 阶段四：进阶跃迁期（1-2年后）

**目标：** 大模型应用开发 → 高级AI工程师 / 技术负责人（25-35K）

- [ ] 能独立设计复杂AI系统架构（多模型协作、高并发服务）
- [ ] 具备团队技术方案评审能力
- [ ] 在某个垂直领域建立技术影响力（技术博客/开源项目/社区分享）
- [ ] 考虑在职考研提升学历天花板

### 考研 vs 回归AI就业

| 维度 | 考研 | 回归AI就业 |
|------|------|-----------|
| 时间成本 | 1年备考 + 2-3年读研 | 1-2个月准备即可投递 |
| 经济压力 | 短期加大（无收入+学费） | 短期缓解（薪资翻倍） |
| 长期收益 | 学历提升，NLP算法岗天花板更高 | 快速积累实战经验，应用层成长快 |
| 风险 | 考不上的沉没成本 | AI科班+落地经验，风险很低 |
| 结论 | 适合经济稳定后的长期规划 | **当前最优选择** |

**建议：** 先回归AI技术岗解决经济压力，工作稳定后再考虑在职考研。两条路不冲突，而且工作经验会让你考研面试更有竞争力。

## 五、面试准备清单

### 项目深度讲解（最重要！）

| 项目 | 必须能回答的问题 |
|------|----------------|
| KG+LLM心理咨询QA | ① 知识图谱构建流程（数据来源→实体抽取→关系构建→入库） |
| | ② BERT NER训练细节（数据标注、训练策略、F1怎么算的） |
| | ③ 为什么同时用KG检索和RAG检索？各自优劣？ |
| | ④ 多模型协作架构设计（Qwen意图识别→KG/RAG→DeepSeek生成） |
| RAG+多Agent简报系统 | ① 向量检索方案选型（用了什么Embedding？为什么？） |
| | ② Agent协作流程怎么设计的？多轮优化具体怎么做？ |
| | ③ 60%自动生成怎么评估的？质量如何保证？ |
| | ④ 遇到过什么工程化挑战？怎么解决的？ |
| 中远海运ETL平台 | ① 四阶段管道设计思路 |
| | ② 异常数据怎么处理？单位换算逻辑？ |

### 八股文重点（大模型应用开发方向）

**必考：**
- Transformer架构（自注意力机制、位置编码、多头注意力、KV Cache）
- BERT原理（MLM、NSP、微调 vs Feature-based、与GPT的区别）
- RAG全流程（文档切分策略、Embedding选型、向量检索、Rerank、生成优化）
- 向量数据库（HNSW/IVF索引原理、Milvus vs Chroma vs Weaviate选型）
- LangChain核心概念（Chain、Agent、Memory、Tool、Callback）

**高频：**
- Prompt Engineering技巧（Few-shot、CoT、Self-consistency）
- Agent设计模式（ReAct、Plan-and-Execute、反思循环）
- Embedding模型对比（BGE、M3E、text2vec、OpenAI Ada）
- 大模型幻觉问题及缓解策略
- 知识图谱基础（三元组、图嵌入、KGQA范式）

### 算法手撕准备

| 类型 | 重点题型 | 目标 |
|------|---------|------|
| 数组/字符串 | 双指针、滑动窗口 | 必须秒杀 |
| 链表 | 反转、合并、环检测 | 必须秒杀 |
| 树 | BFS/DFS、层序遍历 | 熟练 |
| 动态规划 | 背包、最长子序列 | 能做出简单/中等 |
| 哈希表 | 两数之和类 | 必须秒杀 |
