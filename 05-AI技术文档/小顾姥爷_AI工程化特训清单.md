## ⚡️ Day 0 紧急任务：每日工作自动化 (立竿见影)
**核心目标**：解放双手，把每天 2 小时的“找新闻”时间压缩到 5 分钟。

### ✅ 任务 A.1：构建信息源 (RSS + Crawler)
- [ ] **RSSHub 部署**：在本地或服务器部署 RSSHub，把政府网站、公众号变成 RSS 订阅源。
- [ ] **多源聚合**：使用 **Miniflux** 或编写 Python 脚本 (`feedparser`) 聚合所有 RSS 源。

#### 📖 RSS 扫盲与进阶资源
> **RSS (Really Simple Syndication)**：一种信息聚合协议，能让你在一个地方订阅所有网站的更新，无需逐个打开网页。
> **核心逻辑**：网页(HTML) -> RSSHub -> 标准化Feed(XML) -> Python脚本/阅读器。

1.  **入门科普**：
    - [少数派：RSS 入门指南](https://sspai.com/series/26) (通俗易懂，必读)
    - [DIYgod/RSSHub](https://docs.rsshub.app/zh/) (万物皆可 RSS 的神器，文档里有几千个现成的路由)
2.  **必备工具**：
    - **生成源**: [RSSHub Radar](https://github.com/DIYgod/RSSHub-Radar) (浏览器插件，自动发现当前页面的 RSS)
    - **阅读器**: [Fluent Reader](https://github.com/yang991178/fluent-reader) (Windows 上最好看的开源阅读器)
    - **服务端**: [Miniflux](https://miniflux.app/) (极简的 RSS 聚合服务器，推荐部署)
3.  **实战作业**：
    - 安装 RSSHub Radar 插件。
    - 打开“36氪”或“发改委”网站，看插件能否探测到 RSS 源。
    - 如果没有，查阅 RSSHub 文档，看是否支持该网站。

### ✅ 任务 A.2：LLM 智能筛选器 (The Filter)
- [ ] **痛点**：关键词匹配太死板，无关信息太多。
- [ ] **实战**：编写一个 LLM 判别函数。
    - Input: 文章标题 + 摘要
    - Prompt: "你是一个资深政策分析师，请判断这篇文章是否与【低空经济/AI治理】强相关？如果是，请打分(1-10)并简述理由；如果否，返回 0。"
    - Output: JSON `{score: 8, reason: "..."}`
    - **效果**：只看 8 分以上的文章，效率提升 10 倍。

### ✅ 任务 A.3：自动日报生成器
- [ ] **流程**：每天早上 8:00 定时任务 -> 拉取 RSS -> LLM 筛选 -> LLM 总结 -> 生成 Markdown/Word -> 发送邮件。
- [ ] **技术栈**：`Schedule` (定时), `Feedparser` (抓取), `OpenAI/DeepSeek API` (处理), `Jinja2` (生成报告)。

## 🟢 第一阶段：工程化基础与后端开发 (Month 1)
**核心目标**：告别 `print()` 调试，学会写像样的 API 服务。

### ✅ 任务 1.1：现代 Python 开发环境规范 (Day 1-3)
- [ ] **放弃 Jupyter Notebook**：除了做实验，严禁在生产代码中使用 Notebook。
- [ ] **IDE 配置**：熟练使用 VS Code/Cursor/Trae，配置好 Linting (Ruff/Flake8) 和 Formatting (Black)。
- [ ] **类型提示 (Type Hints)**：强制要求所有函数必须写 `def func(a: int) -> str:`。
    - *检查点*：你的代码通过 `mypy --strict` 检查不报错。
- [ ] **包管理**：放弃 `pip freeze > requirements.txt`，学习使用 **Poetry** 或 **uv** 进行依赖管理。

### ✅ 任务 1.2：Web API 开发 (FastAPI) (Day 4-14)
- [ ] **学习 FastAPI**：掌握 Path Parameters, Query Parameters, Pydantic Models。
- [ ] **异步编程**：理解 `async/await`，不要在异步函数里写同步 IO 操作（会被本小姐骂笨蛋的！）。
- [ ] **实战练习**：把你的 "CSV 转 Word" 脚本封装成一个 RESTful API。
    - `POST /convert`: 上传 CSV，返回 Word 文件下载链接。
    - `GET /status/{task_id}`: 查询转换进度。

### ✅ 任务 1.3：数据库设计 (SQL & NoSQL) (Day 15-30)
- [ ] **关系型数据库 (PostgreSQL)**：学习表设计、索引优化。不要什么都塞 JSON！
- [ ] **ORM 框架**：学习 **SQLAlchemy** 或 **Prisma**，别手写 SQL 拼接字符串（太不优雅了！）。
- [ ] **向量数据库**：学习 **pgvector** 或 **ChromaDB**，为 RAG 做准备。

> **🏆 里程碑项目 1：智库文章管理系统 (后端版)**
> *   功能：实现文章的增删改查 API，支持按关键词搜索（SQL），支持按语义相似度搜索（向量）。
> *   要求：代码符合 PEP8 规范，使用 Poetry 管理依赖，有完整的 Swagger 文档。

## 💰 14k+ 薪资冲刺模块：高价值交付物 (面试杀手锏)
**核心目标**：从“写报告的人”变成“构建情报系统的人”。面试时直接演示以下成果。

### ✅ 任务 V.1：交互式情报仪表盘 (Streamlit/Dash)
- [ ] **痛点**：传统的 PDF 报告是静态的，客户无法交互。
- [ ] **实战**：使用 **Streamlit** 构建一个“政策风向监测台”。
    - 功能：地图可视化（各省政策热度）、时间轴（政策演变历史）、词云图（关键词频率）。
    - 技术：Pandas 处理数据，Echarts/Plotly 绘图。

### ✅ 任务 V.2：自动化研报 Agent (LangGraph)
- [ ] **痛点**：人工搜集信息慢，甚至漏掉关键信息。
- [ ] **实战**：构建一个 **"Deep Research Agent"** (仿照 AutoGPT)。
    - 输入："请调研一下最近 3 个月低空经济的政策走向"。
    - 过程：Agent 自动通过 Google/Bing 搜索 -> 抓取 Top 10 网页 -> 总结观点 -> 生成 Markdown 报告。
    - 亮点：展示你对 **LangGraph (状态机)** 和 **Prompt Engineering** 的深度理解。

### ✅ 任务 V.3：垂直领域知识库 (RAG + Knowledge Graph)
- [ ] **痛点**：RAG 经常回答“根据现有文档无法回答”，或者产生幻觉。
- [ ] **实战**：结合你的 **Neo4j** 经验，做一个“图谱增强型 RAG”。
    - 展示：输入“某某政策对房地产的影响”，系统不仅检索文档，还通过图谱展示“政策 -> 影响 -> 房地产”的传导路径。
    - **这是 20k+ 级别的技能点，做出来就能乱杀！**
