## 一、lazygit — Git 终端神器

### 1.1 基本信息

- 版本：v0.59.0
- 位置：`~/.cargo/bin/lazygit.exe`
- 官方文档：`github.com/jesseduffield/lazygit`

### 1.2 启动方式

```bash
# 在当前 Git 仓库中启动
lazygit

# 指定仓库路径
lazygit -p /path/to/repo

# 过滤模式（只看某个文件的历史）
lazygit -f src/main.py

# 直接打开某个面板
lazygit status    # 状态面板
lazygit branch    # 分支面板
lazygit log       # 日志面板
lazygit stash     # 暂存面板
```

### 1.3 界面布局

lazygit 界面分为 5 个面板，用数字键 `1-5` 或 Tab 切换：

```
┌──────────────┬──────────────────────────┐
│ ① Status     │                          │
├──────────────┤   ⑤ Main Panel           │
│ ② Files      │   (diff / commit detail) │
├──────────────┤                          │
│ ③ Branches   │                          │
├──────────────┤                          │
│ ④ Commits    │                          │
├──────────────┤                          │
│ ⑤ Stash      │                          │
└──────────────┴──────────────────────────┘
```

### 1.4 核心快捷键

#### 全局操作

| 按键        | 功能             |
| --------- | -------------- |
| `?`       | 显示当前面板所有快捷键    |
| `h` / `l` | 左右切换面板         |
| `j` / `k` | 上下移动选项         |
| `q`       | 退出 lazygit     |
| `x`       | 打开操作菜单         |
| `@`       | 打开命令日志         |
| `+`       | 展开/收起右侧面板      |
| `:`       | 执行自定义 shell 命令 |

#### Files 面板（文件操作）

| 按键 | 功能 |
|------|------|
| `Space` | 暂存/取消暂存文件（stage/unstage） |
| `a` | 暂存/取消暂存全部文件 |
| `c` | 提交已暂存的更改 |
| `A` | 修改上次提交（amend） |
| `d` | 丢弃文件更改（危险！） |
| `e` | 用编辑器打开文件 |
| `o` | 用系统默认程序打开文件 |
| `i` | 添加到 .gitignore |
| `Enter` | 进入文件查看逐行 diff |
| `S` | 查看暂存选项（stash） |

#### Branches 面板（分支操作）

| 按键 | 功能 |
|------|------|
| `Space` | 切换到选中分支（checkout） |
| `n` | 新建分支 |
| `d` | 删除分支 |
| `r` | 对当前分支 rebase 到选中分支 |
| `M` | 合并选中分支到当前分支 |
| `f` | 快进合并（fast-forward） |
| `R` | 重命名分支 |
| `u` | 设置上游分支 |

#### Commits 面板（提交历史）

| 按键 | 功能 |
|------|------|
| `Space` | 选中/取消选中提交（用于 cherry-pick） |
| `r` | 重写提交信息（reword） |
| `s` | 压缩到上一个提交（squash） |
| `f` | 修正到上一个提交（fixup，不保留信息） |
| `d` | 删除提交 |
| `g` | 重置到此提交（弹出选项：soft/mixed/hard） |
| `p` | cherry-pick 选中的提交 |
| `t` | 给提交打 tag |
| `y` | 复制提交 hash |

### 1.5 日常工作流速查

```
场景1：提交代码
  Files 面板 → Space 选文件 → c 写提交信息 → Enter 确认

场景2：交互式 rebase
  Commits 面板 → 选中提交 → e 进入 rebase 模式
  → 用 s/f/d/r 调整各提交 → 完成后自动应用

场景3：解决合并冲突
  Files 面板 → 选中冲突文件 → Enter 进入
  → 左右面板显示双方更改 → 选择保留哪一方

场景4：暂存当前工作
  Files 面板 → S → 选择 stash 选项 → 输入描述
  恢复：Stash 面板 → Space 应用 / g 弹出
```

### 1.6 推荐配置

配置文件位置：`%APPDATA%\lazygit\config.yml`

```yaml
# 推荐修改项
gui:
  showIcons: true           # 开启图标（需要 Nerd Font）
  nerdFontsVersion: "3"     # Nerd Font v3
  language: "zh-CN"         # 中文界面
  mouseEvents: true         # 鼠标支持
  theme:
    activeBorderColor:
      - "#89b4fa"           # Catppuccin 风格高亮
      - bold
os:
  editPreset: "nvim"        # 或 "code" 用 VS Code
```

## 二、yazi — 极速终端文件管理器

### 2.1 基本信息

- 版本：v26.1.22
- 位置：`~/.cargo/bin/yazi.exe`（主程序）+ `ya.exe`（CLI 辅助）
- 官方文档：`yazi-rs.github.io`

### 2.2 启动方式

```bash
# 在当前目录启动
yazi

# 打开指定目录
yazi /path/to/dir

# 退出时将当前目录写入文件（配合 shell 集成）
yazi --cwd-file /tmp/yazi-cwd

# 清除缓存（预览缩略图等）
yazi --clear-cache
```

### 2.3 界面布局

```
┌──────────────┬──────────────────┬──────────────────┐
│  Parent Dir  │  Current Dir     │  Preview Panel   │
│  (上级目录)   │  (当前目录)       │  (文件预览)       │
│              │                  │                  │
│  h ← 返回    │  文件列表         │  代码高亮/图片/   │
│              │  j/k 上下移动     │  PDF/视频缩略图   │
│              │  l → 进入/打开    │                  │
└──────────────┴──────────────────┴──────────────────┘
                    底部状态栏
```

### 2.4 核心快捷键

#### 导航操作

| 按键 | 功能 |
|------|------|
| `h` | 返回上级目录 |
| `l` / `Enter` | 进入目录 / 打开文件 |
| `j` / `k` | 上下移动光标 |
| `J` / `K` | 向下/向上移动半页 |
| `gg` | 跳到第一个文件 |
| `G` | 跳到最后一个文件 |
| `~` | 跳到 Home 目录 |
| `-` | 跳到上次访问的目录 |
| `z` | 使用 zoxide 智能跳转 |

#### 文件操作

| 按键  | 功能                    |
| --- | --------------------- |
| `o` | 用系统默认程序打开             |
| `O` | 用交互式方式选择打开程序          |
| `y` | 复制（yank）选中文件          |
| `x` | 剪切选中文件                |
| `p` | 粘贴到当前目录               |
| `P` | 粘贴并覆盖同名文件             |
| `d` | 移到回收站（trash）          |
| `D` | 永久删除（危险！）             |
| `a` | 新建文件/目录（末尾加 `/` 创建目录） |
| `r` | 重命名                   |
| `.` | 显示/隐藏 隐藏文件            |

#### 选择与搜索

| 按键 | 功能 |
|------|------|
| `Space` | 选中/取消选中当前文件 |
| `v` | 进入可视选择模式 |
| `V` | 反选所有文件 |
| `Esc` | 取消选择 / 退出模式 |
| `f` | 过滤当前目录（实时筛选） |

#### 标签页与排序

| `1-9` | 切换到第 N 个标签页 |
| `[` / `]` | 切换上/下一个标签页 |
| `,` | 切换排序方式（名称/大小/日期） |
| `w` | 切换任务管理器（查看复制进度等） |
| `q` | 关闭当前标签页 / 退出 yazi |

### 2.5 Shell 集成（退出后跳转到浏览目录）

在 `~/.bashrc` 或 `~/.bash_profile` 中添加：

```bash
function y() {
  local tmp="$(mktemp -t "yazi-cwd.XXXXXX")" cwd
  yazi "$@" --cwd-file="$tmp"
  if cwd="$(command cat -- "$tmp")" && [ -n "$cwd" ] && [ "$cwd" != "$PWD" ]; then
    builtin cd -- "$cwd"
  fi
  rm -f -- "$tmp"
}
```

之后用 `y` 代替 `yazi` 启动，退出时自动 cd 到最后浏览的目录。

### 2.6 插件管理

```bash
# 安装插件（以 lazygit 集成为例）
ya pkg add yazi-rs/plugins:lazygit

# 更新所有插件
ya pkg upgrade

# 列出已安装插件
ya pkg list
```

推荐插件：
- `lazygit.yazi` — 在 yazi 中按快捷键呼出 lazygit
- `zoxide.yazi` — 智能目录跳转集成

### 2.7 配置文件

配置目录：`%APPDATA%\yazi\`

```
%APPDATA%\yazi\
├── yazi.toml      # 主配置（文件管理行为）
├── keymap.toml    # 快捷键自定义
├── theme.toml     # 主题配色
└── init.lua       # Lua 初始化脚本
```

## 三、DeepWiki MCP — 仓库 Wiki 即时查询

### 3.1 基本信息

- 集成方式：MCP 工具 `deepwiki_fetch`（Claude Code 内置）
- 在线访问：`deepwiki.com/owner/repo`
- 无需额外安装，对话中直接调用

### 3.2 使用场景

| 场景 | 说明 |
|------|------|
| 快速了解陌生仓库 | 输入仓库名，秒出架构文档 |
| 学习开源项目 | 自动生成 Mermaid 架构图 + 模块说明 |
| 技术选型调研 | 对比多个仓库的设计思路 |
| 代码审查辅助 | 理解项目结构后再看具体代码 |

### 3.3 在 Claude Code 中调用

在对话中直接说：

```
帮我查一下 sxyazi/yazi 这个仓库的架构
```

Claude Code 会自动调用 `deepwiki_fetch` 工具，返回：
- 项目概览与核心模块说明
- 代码架构图（Mermaid 格式）
- 关键文件与依赖关系
- 各子系统的详细文档目录

### 3.4 支持的输入格式

```
# 以下格式都可以
owner/repo          → sxyazi/yazi
完整 GitHub URL     → https://github.com/sxyazi/yazi
两个词              → sxyazi yazi
单个关键词          → yazi（模糊匹配）
```

### 3.5 参数说明

| 参数 | 值 | 说明 |
|------|-----|------|
| `maxDepth` | `0` | 只获取首页概览（快速） |
| `maxDepth` | `1` | 获取所有子页面（完整但慢） |
| `mode` | `aggregate` | 聚合为单个文档（默认） |
| `mode` | `pages` | 按页面分别返回 |

## 四、CCometixLine — Claude Code 状态栏

### 4.1 基本信息

- 版本：v1.1.1
- 位置：`~/.cargo/bin/ccline.exe`
- 状态：已配置使用中

### 4.2 常用命令

```bash
# 进入 TUI 交互式配置界面
ccline -c

# 初始化配置文件
ccline --init

# 查看当前配置
ccline --print

# 检查配置是否正确
ccline --check

# 切换主题
ccline -t <theme_name>

# 检查更新
ccline -u

# 修补 Claude Code cli.js（禁用上下文警告弹窗）
ccline --patch
```

### 4.3 显示内容

状态栏实时显示：
- 当前使用的模型名称
- 工作目录路径
- Git 分支名 + 状态（clean/dirty）
- 上下文窗口使用量（百分比）
- Token 消耗追踪

## 五、Nerd Font 配置指南

### 5.1 已安装字体

- 字体族：JetBrainsMono Nerd Font v3.4.0
- 位置：`%LOCALAPPDATA%\Microsoft\Windows\Fonts\`
- 字体数量：96 个 TTF 变体（Regular/Bold/Italic/Mono/Propo 等）

### 5.2 三种字体变体的区别

| 变体 | 名称 | 特点 | 适用场景 |
|------|------|------|---------|
| Nerd Font | `JetBrainsMonoNerdFont` | 图标等宽2格 | 终端（推荐） |
| Mono | `JetBrainsMonoNerdFontMono` | 图标压缩为1格 | 严格等宽场景 |
| Propo | `JetBrainsMonoNerdFontPropo` | 图标按比例宽度 | 编辑器/IDE |

### 5.3 各终端配置方法

#### Windows Terminal

设置 → 配置文件 → 默认值 → 外观 → 字体：

```json
// settings.json 中对应片段
"profiles": {
  "defaults": {
    "font": {
      "face": "JetBrainsMono Nerd Font",
      "size": 13
    }
  }
}
```

#### VS Code 终端

设置中搜索 `terminal.integrated.fontFamily`：

```json
"terminal.integrated.fontFamily": "JetBrainsMono Nerd Font"
```

#### Git Bash / MINGW

编辑 `~/.minttyrc`：

```
Font=JetBrainsMono Nerd Font
FontSize=12
```

### 5.4 安装后注意事项

- 安装/更换字体后需要**重启终端**才能生效
- 如果图标显示为方块 `□`，检查终端是否正确设置了 Nerd Font 字体
- lazygit 和 yazi 的图标功能都依赖 Nerd Font，务必先配好字体再使用
