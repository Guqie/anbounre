# designprompts.dev 使用指南

> AI 设计风格提示词库 — 让 AI 生成美观一致的界面

**官网**: https://www.designprompts.dev/

---

## 一、项目简介

### 核心定位

**designprompts.dev** — AI 驱动的设计风格探索器

- 提供 30 种精选设计风格的标准化提示词
- 支持任何 AI 助手（Claude、GPT、Gemini、Cursor 等）
- 即复制即用，快速生成美观一致的界面
- 解决设计风格标准化表达的痛点

### 解决的问题

**传统方式的痛点**:
- "我想要一个现代感的界面" — 太模糊
- "带点科技感的深色主题" — AI 理解不一致
- "类似 Apple 官网的风格" — 描述不精确

**designprompts 的解决方案**:
- 标准化的风格名称（如 "Modern Dark"）
- 详细的风格描述提示词
- 一致的设计语言和视觉规范

---

## 二、核心功能

### 1. 30 种设计风格库

**覆盖范围**:
- 极简主义（Monochrome、Swiss Minimalist、Minimal Dark）
- 现代风格（Modern Dark、Material Design、Flat Design）
- 复古风格（Retro、Art Deco、Bauhaus）
- 科技风格（Cyberpunk、Terminal、Web3）
- 企业风格（Enterprise、Professional、SaaS）
- 创意风格（Neo Brutalism、Maximalism、Playful Geometric）

### 2. 智能筛选系统

**按模式筛选**:
- All — 显示所有风格
- Light — 仅显示浅色主题
- Dark — 仅显示深色主题

**按字体类型筛选**:
- All — 显示所有字体类型
- Sans — 无衬线字体风格
- Serif — 衬线字体风格
- Mono — 等宽字体风格

### 3. 即用型提示词

每个风格都包含：
- **风格名称**: 简洁的标识符
- **详细描述**: 完整的设计语言描述
- **视觉特征**: 色彩、排版、布局规范
- **情感定位**: 传达的氛围和感觉

### 4. 实时预览

- 点击任意风格卡片
- 查看完整的设计示例
- 理解风格的实际效果

## 四、使用方法

### 基础使用流程

**Step 1: 选择风格**
```
1. 访问 https://www.designprompts.dev/
2. 浏览 30 种设计风格
3. 使用筛选器缩小范围（Light/Dark, Sans/Serif/Mono）
4. 点击感兴趣的风格卡片
```

**Step 2: 获取提示词**
```
1. 在风格详情页查看完整描述
2. 点击 "Get Prompt" 按钮
3. 提示词自动复制到剪贴板
```

**Step 3: 应用到 AI 助手**
```
1. 打开 Claude Code / Cursor / 任何 AI 助手
2. 粘贴提示词
3. 添加具体需求（如 "创建一个登录页面"）
4. AI 生成对应风格的代码
```

### 提示词使用技巧

**基础用法**:
```
使用 Monochrome 风格，创建一个产品展示页面
```

**进阶用法**:
```
使用 Cyberpunk 风格，创建一个仪表盘界面，
包含数据可视化图表、实时状态监控和操作按钮
```

**组合用法**:
```
整体使用 Material Design 风格，
但导航栏采用 Bold Typography 风格的大标题
```

## 五、实战案例

### 案例 1: 创建 SaaS 产品落地页

**需求**: 为 SaaS 产品创建专业的落地页

**操作步骤**:
1. 选择 "SaaS" 风格（Light, Sans）
2. 获取提示词
3. 在 Claude Code 中输入：
   ```
   使用 SaaS 风格，创建一个产品落地页，包含：
   - Hero 区域（标题、副标题、CTA 按钮）
   - 功能特性展示（3列布局）
   - 客户评价
   - 定价表格
   ```

**预期效果**:
- 清晰的层级结构
- 专业的商务感
- 高转化率的设计

### 案例 2: 创建开发者工具文档站

**需求**: 为开源项目创建文档网站

**操作步骤**:
1. 选择 "Terminal" 风格（Dark, Mono）

   ```

   - 侧边栏导航
   ```

**预期效果**:

## 六、与 AI 工具集成

### Claude Code 集成

**方式 1: 直接使用**
```
在聊天框中输入：
"使用 [风格名称] 风格，创建 [具体需求]"
```

**方式 2: 配置到 CLAUDE.md**
```markdown
# 设计风格偏好

默认使用 designprompts.dev 的设计风格：
- 企业项目：Enterprise 或 Professional
- 创意项目：Neo Brutalism 或 Playful Geometric
- 技术项目：Terminal 或 Cyberpunk
```

### Cursor 集成

**方式 1: Composer 模式**
```
在 Composer 中输入：
"参考 designprompts.dev 的 [风格名称] 风格"
```

**方式 2: Rules for AI**
```
在 .cursorrules 中添加：
"UI 设计遵循 designprompts.dev 的 Material Design 风格"
```

### 其他 AI 助手

**ChatGPT / GPT-4**:
```
直接粘贴提示词 + 具体需求
```

**Gemini**:
```
特别适合前端 UI 设计，配合 designprompts 效果最佳
```

## 七、最佳实践

### 1. 风格选择原则

**根据项目类型选择**:
- 企业官网 → Enterprise, Professional
- SaaS 产品 → SaaS, Material Design
- 开发者工具 → Terminal, Modern Dark
- 创意作品集 → Neo Brutalism, Maximalism
- 电商平台 → Flat Design, Claymorphism

**根据目标用户选择**:
- 技术人员 → Terminal, Cyberpunk
- 商务人士 → Enterprise, Professional
- 年轻用户 → Playful Geometric, Vaporwave
- 高端用户 → Luxury, Art Deco

### 2. 提示词优化技巧

**明确具体需求**:
```
❌ 不好：使用 Modern Dark 风格创建一个页面
✅ 好：使用 Modern Dark 风格创建一个登录页面，
      包含邮箱输入框、密码输入框、记住我选项和登录按钮
```

**组合多个风格**:
```
整体使用 Material Design 风格，
但 Hero 区域采用 Bold Typography 的大标题设计
```

**指定技术栈**:
```
使用 Terminal 风格，用 React + Tailwind CSS 创建组件
```

### 3. 常见问题解决

**问题 1: AI 生成的风格不够准确**
- 解决：复制完整的风格描述，而非仅风格名称
- 解决：添加更多视觉细节描述

**问题 2: 风格与品牌不符**
- 解决：在提示词中添加品牌色彩要求
- 解决：选择更接近品牌调性的风格

**问题 3: 多页面风格不一致**
- 解决：在项目开始时确定统一风格
- 解决：在 CLAUDE.md 中配置默认风格
