```

│ exa MCP  │ RSS 订阅 │ 网页爬虫 │ Google   │ Playwright   │
│ 搜索     │          │ (news-   │ News API │ 浏览器自动化  │
│          │          │ workflow)│          │              │
```

## 三、时效性控制机制

新闻的价值随时间衰减，必须分级管理：

### 时效性分级

| 级别 | 时间窗口 | 适用内容 | 对应渠道 |
|------|---------|---------|---------|
| **S级·即时** | 当日 | 突发政策、重大事件、部委发文 | RSS 订阅（实时推送） |
| **A级·当日** | 24小时内 | 产业动态、企业新闻、数据发布 | Google News API（today过滤） |
| **B级·本周** | 7天内 | 行业分析、研报、深度报道 | exa 搜索 + 爬虫 |
| **C级·本月** | 30天内 | 政策解读、趋势研判、专题汇编 | exa 搜索 |

### 各渠道的时效性控制方式

| 渠道 | 时效控制手段 | 精度 |
|------|------------|------|
| RSS 订阅 | 发布即推送，天然实时 | ⭐⭐⭐⭐⭐ 分钟级 |
| Google News API | `period` 参数：`1d`/`7d`/`1m` | ⭐⭐⭐⭐ 天级 |
| exa MCP | 搜索词加年份；无精确时间过滤 | ⭐⭐⭐ 月级 |
| 网页爬虫 | 按发布日期过滤（代码中实现） | ⭐⭐⭐⭐ 天级 |
| Playwright | 实时抓取，手动触发 | ⭐⭐⭐⭐⭐ 实时 |

### 结论：日常监控必须以 RSS 为主力

exa 虽然质量最高，但**不适合做日常新闻监控**——它返回的是"最相关"而非"最新"。

**正确的组合策略：**
```
日常监控（每日）：RSS 订阅 + Google News API
    → 保证时效性，不漏重要新闻

深度补充（每周）：exa 搜索
    → 补充高质量深度内容、政策原文

专题调研（按需）：exa + Playwright
    → 针对特定主题深挖
```

## 五、统一管理架构

### 信源总量规划

| 层级 | 数量 | 更新频率 | 作用 |
|------|------|---------|------|
| 第一层·关键词聚合 | 9个 Bing RSS | 小时级 | 全领域覆盖，不漏热点 |
| 第二层·媒体直订 | 8个 RSSHub | 实时 | 权威信源，政策+快讯 |
| 第三层·exa 搜索 | 按需 | 手动/每周 | 深度内容，专题调研 |
| 第四层·爬虫采集 | 13个信源 | 每日定时 | 全文提取，批量处理 |
| **合计** | **~30个信源** | — | — |

### 每日信息流水线

```
06:00  定时任务自动运行
       ├── RSS 拉取（第一层+第二层）→ 聚合到阅读器
       └── news-workflow 爬虫（第四层）→ 输出到 data/raw/

10:00  人工审阅（你的工作时间）
       ├── 阅读器中快速浏览 RSS（15分钟）
       ├── 筛选 3-5 条高价值信息
       └── 写三行摘要

按需   exa 深度搜索（第三层）
       ├── 专题汇编时搜索
       └── 政策发布时追踪原文
```

## 六、与现有系统的对接

### 数据流向：采集 → 处理 → 归档

```
信源（30个）
  │
  ├─ RSS（17个）──→ Fluent Reader / engine RSS聚合
  ├─ 爬虫（13个）─→ news-workflow → data/raw/
  └─ exa（按需）──→ Claude Code 会话内直接使用
  │
  ▼
engine SQLite 队列（统一入口）
  │
  ▼
python main.py process
  ├── AI 三行摘要
  ├── PIDST 分类
  └── 质量评分
  │
  ▼
Obsidian 知识库归档
  ├── 02-政策与宏观库/
  ├── 03-行业研究库/
  └── 00-每日工作区/03-三行摘要收集
```

### 需要打通的断裂点

| 断裂点 | 现状 | 修复方案 |
|--------|------|---------|
| RSS → engine | feeds.json 只有2个源 | 扩展到17个源 |
| news-workflow → engine | 完全断开 | 爬虫输出导入 SQLite 队列 |
| engine → Obsidian | 部分可用 | `process` 命令已支持，需验证 |
