## 三、测试结果

### exa MCP 测试结果

| 领域 | 可用性 | 信源质量 | 备注 |
|------|--------|----------|------|
| 智能驾驶 | ✅ | 国务院/部委/央媒 | 可搜到政策原文 |
| 人形机器人 | ✅ | 权威财经/科技媒体 | 产业动态丰富 |
| 低空经济 | ✅ | 央媒/行业网站 | 政策 + 企业新闻 |
| 商业航天 | ✅ | 权威信源 | 深度内容多 |
| 氢能发展 | ✅ | 部委/行业媒体 | 政策密集 |
| 人工智能 | ✅ | 全信源覆盖 | 信息量最大 |
| 低碳经济 | ✅ | 权威信源 | 双碳政策丰富 |
| 战略新兴 | ✅ | 财经/部委 | 产业覆盖全 |
| 热点风口 | ✅ | 综合信源 | 趋势研判多 |

### exa MCP 的时效性问题

**核心问题：** exa 返回的是"相关性最高"的结果，不是"最新"的结果。

| 搜索策略 | 时效性控制 | 精度 |
|----------|-----------|------|
| 搜索词加"2026" | 保证今年内容 | ⭐⭐⭐ 月级 |
| 搜索词加"2 月" | 保证本月内容 | ⭐⭐⭐ 月级 |
| 无时间过滤 | 无法控制 | ⭐⭐ 不可控 |

**结论：** exa 不适合做日常新闻监控，但适合深度搜索和专题调研。

## 五、RSS MCP 安装建议

**首选推荐：** `@missionsquad/mcp-rss`

**安装方式：** 在 Claude Code 配置文件中添加

```json
{
  "mcpServers": {
    "mcp-rss": {
      "command": "npx",
      "args": ["@missionsquad/mcp-rss"]
    }
  }
}
```

**核心能力：**
- RSS/Atom 解析
- 内存缓存（避免重复拉取）
- 批量获取（一次拉取多个源）
- 全文搜索（在 RSS 内容中搜索关键词）
- OPML 导出（方便备份和迁移）
