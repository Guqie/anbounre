## 一、产品概述

### 1.1 产品定位

面向中车研究院的 **月度产业情报刊物**，围绕"轨道交通装备转型升级、战新产业发展壮大、未来产业培育布局"三大战略方向，提供"政策—产业—技术—竞争"全链条情报支撑。

### 1.2 核心价值主张

| 维度 | 价值 |
|------|------|
| 战略决策 | 精准传递政策导向，研判产业机遇与风险 |
| 技术研发 | 追踪前沿技术突破，提供研发方向参考 |
| 市场布局 | 洞察竞争格局变化，支撑合作与投资决策 |

### 1.3 产品基本信息

| 项目 | 说明 |
|------|------|
| 产品名称 | 战新产业与未来产业月报 |
| 刊期 | 月刊 |
| 覆盖周期 | 每月15日 — 次月15日 |
| 出品方 | 安邦信息数据研究中心 |
| 交付格式 | Word 文档（.docx） |
| 期刊编号 | YYYY年第N期（如"2026年第1期"） |

## 二、目标用户与使用场景

### 2.1 目标用户

| 用户角色 | 核心诉求 | 阅读重点 |
|---------|---------|---------|
| 中车研究院管理层 | 战略方向研判、投资决策参考 | 摘要、政策解读、竞争格局 |
| 技术研发人员 | 前沿技术跟踪、研发方向启发 | 技术前沿、专利标准、科研转化 |
| 市场与战略部门 | 市场趋势洞察、竞争对手监测 | 产业洞察、企业动态、合作并购 |
| 产业链管理人员 | 供应链安全、上下游动态 | 产业链追踪、价格波动、产能布局 |

### 2.2 典型使用场景

1. **月度战略会议**：管理层通过摘要快速掌握本月产业全貌，结合政策解读研判战略方向
2. **技术选型论证**：研发团队参考技术前沿栏目，了解行业技术迭代趋势与竞品技术路线
3. **投资并购评估**：战略部门通过竞争格局与企业动态栏目，评估潜在合作/并购标的
4. **供应链风险预警**：采购部门关注产业链动态追踪，提前识别供应链风险

### 3.2 栏目详细规格

#### 栏目一：政策导航与战略解读

> 核心功能：聚焦国内外战新产业与未来产业顶层设计，精准传递政策导向，解读政策对中车核心业务及延伸领域的影响。

| 二级栏目       | 内容范围                                                | 信息来源                |
| ---------- | --------------------------------------------------- | ------------------- |
| （一）清洁能源装备  | 国家层面（国务院、发改委、工信部、国资委等）发布的新能源、智能制造相关政策；地方配套政策与产业扶持措施 | 政府官网、中国新闻网、光明日报、人民网 |
| （二）数智及未来产业 | AI+制造、工业互联网、低空经济、海工装备等领域政策规划与行动方案                   | 新华网、科技日报、人民邮电报      |

**单条资讯结构（基于成稿实际格式）：**

```
### N. 资讯标题

[正文段落1：事件概述]

[正文段落2-3：核心内容，关键信息加粗标注]

信息来源：[来源名称，日期](URL)

政策原文：[文件名称，日期](URL)  ← 如有政策原文则附上

#### 政策解读：解读标题  ← 重要政策附带解读板块
[解读正文，含多个小标题分段]
信息来源：[来源名称，日期](URL)
```

#### 栏目二：产业全景与细分洞察

> 核心功能：呈现战新产业与未来产业整体发展态势，量化市场规模与增长趋势，明确产业发展痛点与机会点。

| 二级栏目 | 内容范围 |
|---------|---------|
| （一）清洁能源装备 | 装机容量数据、出口增长、储能市场规模、风电叶片技术趋势、产业链供需与价格波动 |
| （二）数智及未来产业 | 工业机器人产量、低空经济市场规模、深海科技订单、节能环保企业数量、产业景气度 |

**内容特征（基于成稿分析）：**
- 每条资讯以数据驱动，优先呈现量化指标（装机容量、产量增速、市场规模等）
- 行业趋势类资讯篇幅较长（800-1500字），含多个加粗小标题分段
- 数据类快讯篇幅较短（200-400字），突出核心数字

#### 栏目三：技术前沿与创新突破

> 核心功能：聚焦核心技术迭代趋势，收录关键技术突破与创新成果，为中车技术研发提供灵感与方向参考。

#### 栏目四：竞争格局与企业动态

> 核心功能：追踪国内外头部企业战略布局、业务拓展、合作并购动态，为中车市场竞争与合作决策提供支撑。

## 六、编辑规范与排版要求

### 6.1 内容编辑红线

| 序号 | 规则 | 说明 |
|------|------|------|
| 1 | 删除人名/专家名 | 删除所有人名，同时删除"专家、认为、分析、表示、介绍"等引述词 |
| 2 | 标题与首句去重 | 资讯第一句与标题意思重复时，首句不加粗 |
| 3 | 标题序号格式 | 序号后加一个空格（如 `1. 标题` 而非 `1.标题`） |
| 4 | 排除中车新闻 | 不收录中车自身相关新闻 |
| 5 | 字体规范 | 数字、英文统一使用 Times New Roman 字体 |

### 6.2 排版规范（基于成稿逆向提取）

| 元素 | 规范 |
|------|------|
| 一级栏目标题 | `# 一、栏目名称`（Word 中为一级标题） |
| 二级分类标题 | `## （一）清洁能源装备` / `## （二）数智及未来产业` |
| 资讯标题 | `### N. 资讯标题`（N 为该分类下的序号） |
| 政策解读标题 | `#### 政策解读：解读标题` |
| 关键信息 | 使用 **加粗** 标注核心数据、政策要点、趋势判断 |
| 信息来源 | `信息来源：[来源名称，YYYYMMDD](URL)` |
| 政策原文 | `政策原文：[文件名称，YYYYMMDD](URL)` |

### 6.3 加粗策略（基于成稿分析）

加粗用于引导读者快速抓取关键信息，具体规则：

- **政策发布日期与文件名称**：如"1月9日，工信部等五部门联合印发《工业绿色微电网建设与应用指南》"
- **核心量化数据**：如"全国累计发电装机容量37.9亿千瓦"
- **趋势判断与结论性表述**：如"行业将呈现'大电芯、液冷化、长时化'的发展特点"
- **分点论述的序号引导**：如"一是…二是…三是…"
- ⚠️ 资讯首句与标题意思重复时，首句不加粗

## 七、生产流程与交付节奏

### 7.1 五阶段生产流程

```
阶段一：信息采集（D-30 ~ D-15）
  → 按信源清单 + 关键词库逐领域采集，目标 200-250 条原始素材

阶段二：筛选分类（D-15 ~ D-12）
  → 执行筛选标准（删中车/删专家引述/删无关/去重）
  → 按四大栏目 × 两大领域分类，筛选后约 150-180 条

阶段三：内容编写（D-12 ~ D-7）
  → 按栏目撰写正文，重要政策附解读
  → 最终精选入稿约 50-70 条
  → 撰写摘要（500-800字）

阶段四：格式审核（D-7 ~ D-3）
  → 编辑红线 5 条逐项自检
  → 排版定稿 + 目录页码对齐
  → 页数确认 ≥ 35 页

阶段五：终稿交付（D-3 ~ D-0）
  → 通读全文检查逻辑性与数据准确性
  → 生成终稿 Word 文档交付客户

D = 每月月底（如1月30日交付覆盖12.15—1.15的月报）
```

### 7.2 单条资讯生产流程

```
发现信源 → 判断是否属于关注领域 → 提取核心信息 → 撰写正文
→ 加粗标注关键信息 → 附信息来源链接 → 重要政策附解读板块
→ 编辑规范自检（5条红线）→ 归入对应栏目
```

### 7.3 摘要撰写规范

摘要位于封面之后、目录之前，约 500-800 字，结构如下：
