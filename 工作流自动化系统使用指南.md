## 系统简介

**智库工作流自动化系统**是一个基于 Python 的智能工作流管理工具，专为智库研究员设计。系统集成了 AI 分析能力和数据库管理，实现了从信息采集到知识沉淀的**端到端自动化闭环**。

### 🎯 系统目标

以"材料队列"为核心，实现**"采集 → 入库 → 自动处理 → 结构化归档 → 索引更新"**的完整闭环，支持政府库/行业库批量导入，自动生成三行摘要、PIDST 分类，并沉淀到知识库。

### ✨ 核心特性

- 🚀 **端到端自动化**：从材料导入到知识沉淀的全流程自动化
- 💾 **数据库管理**：SQLite 材料队列，支持状态追踪和批量处理
- 📥 **批量导入**：支持 CSV/JSON 格式，兼容中英文列名
- 🤖 **AI 驱动分析**：自动生成三行摘要、提炼洞察、构建证据链
- 📚 **智能知识管理**：自动分类归档、标签生成、全文检索
- ⏰ **任务自动化**：每日任务模板、工作流程提醒、复盘报告
- 🎯 **PIDST 框架**：基于五维框架的信息分类和管理
- 🔗 **知识关联**：自动发现知识之间的关联关系

## 快速开始

### 1. 安装依赖

```bash
# 进入项目目录
cd "D:\桌面\无敌战警1.0"

# 安装 Python 依赖包
pip install -r requirements.txt
```

### 2. 配置 API Key（推荐使用 OpenRouter）

编辑 `config/config.yaml` 文件，配置你的 AI API Key：

```yaml
ai:
  # 使用 OpenRouter（推荐！支持多种模型）
  provider: "openrouter"

  openrouter:
    api_key: "sk-or-v1-xxxxx..."  # 🔑 填写你的 OpenRouter API Key
    model: "anthropic/claude-3.5-sonnet"  # 🤖 选择模型
```

**支持的 AI 提供商：**
- **OpenRouter**（⭐ 推荐）：统一接口，支持多种模型，性价比高
- **OpenAI**：需要 OpenAI API Key
- **Anthropic**：需要 Anthropic API Key
- **Local**：使用本地模型（如 Ollama），无需 API Key

**详细配置指南：** 查看 `config/OpenRouter配置指南.md`

### 3. 初始化系统

```bash
python main.py init
```

这将创建必要的目录结构、数据库并检查配置。

### 4. 开始使用

#### 传统方式（单条处理）

```bash
# 生成今日任务模板
python main.py daily

# 分析材料并生成三行摘要
python main.py analyze --content "材料内容" --source "来源" --archive

# 提炼洞察
python main.py insight --content "信息汇总" --archive

# 查看帮助
python main.py --help
```

#### 自动化方式（批量处理）⭐ 新功能

```bash
# 1. 批量导入政府库材料（CSV/JSON）
python main.py import-gov --file "政府库数据.csv"

# 2. 批量导入行业库材料
python main.py import-ind --file "行业库数据.csv"

# 3. 自动处理队列中的所有材料
python main.py process

# 4. 查看队列状态
python main.py queue-status
```

## 核心功能

### 1. AI 分析引擎

#### 三行摘要生成

自动将复杂信息提炼为结构化的三行摘要：

```bash
# 从文件分析
python main.py analyze --file "待处理材料.md" --archive

# 从文本分析
python main.py analyze \
  --content "材料内容" \
  --source "来源" \
  --title "材料标题" \
  --archive
```

**输出格式：**
```markdown
### 材料标题
**1. What happened：** 发生了什么
**2. So what：** 意味着什么
**3. Now what：** 下一步如何
**标签：** Policy/Industry/Data
**时间：** 2025-12-03
**来源：** 官方网站
```

#### 观点提炼

从信息中提炼深刻洞察：

```bash
python main.py insight \
  --file "信息汇总.md" \
  --min-insights 3 \
  --topic "AI+医疗" \
  --archive
```

**输出格式：**
```markdown
### 洞察 1：核心观点
**依据：** 支撑证据
**逻辑链：** 推理过程
**趋势：** 未来走向
**影响：** 可能影响
```

#### 证据链构建

按照 F-D-P-T-S 五维框架构建完整证据链：

```bash
python main.py evidence \
  --topic "医疗AI发展趋势" \
  --file "相关信息.md" \
  --archive
```

**输出维度：**
- **F (Facts)**：客观事实
- **D (Data)**：量化数据
- **P (Policy)**：政策文件
- **T (Trend)**：发展趋势
- **S (Signal)**：弱信号

#### 趋势分析

对特定领域进行趋势判断：

```bash
python main.py trend \
  --domain "AI+教育" \
  --file "行业信息.md" \
  --timeframe "未来3-5年"
```

## 端到端自动化

### 🎯 核心理念

本小姐设计的端到端自动化系统，以"材料队列"为核心，实现从信息采集到知识沉淀的完整闭环，显著降低重复劳动，提升工作效率！(￣▽￣)／

### 💾 数据库管理

#### 材料队列数据库

系统使用 SQLite 数据库 (`data/materials.db`) 管理材料队列，支持状态追踪和批量处理。

**数据模型：**

| 字段 | 类型 | 说明 |
|-----|------|------|
| `id` | INTEGER | 主键 |
| `type` | TEXT | 材料类型（government/industry/topic） |
| `title` | TEXT | 材料标题 |
| `content` | TEXT | 材料内容 |
| `source` | TEXT | 来源 |
| `url` | TEXT | 链接 |
| `agency` | TEXT | 发布机构（政府库） |
| `doc_number` | TEXT | 文号（政府库） |
| `subject` | TEXT | 主题（行业库） |
| `event_type` | TEXT | 事件类型（行业库） |
| `key_data` | TEXT | 关键数据（行业库） |
| `date` | TEXT | 日期 |
| `status` | TEXT | 状态（pending/processing/completed/error） |
| `result_json` | TEXT | 处理结果（JSON） |
| `error_msg` | TEXT | 错误信息 |
| `created_at` | TIMESTAMP | 创建时间 |
| `updated_at` | TIMESTAMP | 更新时间 |

**状态机：**

```
pending → processing → completed
                    ↘ error
```

#### 查看队列状态

```bash
# 查看队列统计
python main.py queue-status

# 输出示例：
# 📊 材料队列状态：
# ⏳ 待处理: 15
# 🔄 处理中: 2
# ✅ 已完成: 128
# ❌ 失败: 3
```

### 🔄 队列处理

#### 自动处理流程

```bash
# 处理队列中的所有待处理材料
python main.py process

# 指定处理数量
python main.py process --limit 10

# 指定材料类型
python main.py process --type government
```

**处理流程：**

1. **读取队列**：从数据库读取 `pending` 状态的材料
2. **AI 分析**：调用 AI 生成三行摘要
3. **PIDST 分类**：自动分类到五维框架
4. **结构化归档**：按库分类存储 Markdown
5. **更新索引**：更新知识库索引
6. **状态更新**：标记为 `completed` 或 `error`

**输出示例：**

```
🔄 正在处理材料队列...

[1/15] 处理中: 关于推进AI发展的通知
  ✅ 三行摘要生成完成
  ✅ PIDST 分类: Policy
  ✅ 已归档到: 02-政策于宏观库/2025-01-15-关于推进AI发展的通知.md

[2/15] 处理中: 某公司发布AI新产品
  ✅ 三行摘要生成完成
  ✅ PIDST 分类: Industry
  ✅ 已归档到: 03-行业研究库/2025-01-20-某公司发布AI新产品.md

...

✨ 处理完成！
  ✅ 成功: 13
  ❌ 失败: 2
  ⏱️  耗时: 3分25秒
```

### ⚡ 性能优化建议

**批量导入：**
- 单次导入建议不超过 100 条记录
- 大批量数据建议分批导入

**队列处理：**
- 使用 `--limit` 参数控制单次处理数量
- 避免在高峰期大量调用 AI API

**成本控制：**
- 优先使用性价比高的模型（如 `gpt-3.5-turbo`）
- 使用 OpenRouter 统一管理多个模型

## 常见问题

### Q1: 如何配置本地 AI 模型？

**A:** 编辑 `config/config.yaml`：

```yaml
ai:
  provider: "local"
  local_model:
    endpoint: "http://localhost:11434"  # Ollama 端点
    model_name: "llama2"                # 模型名称
```

然后确保 Ollama 服务正在运行：

```bash
ollama serve
```

### Q2: API Key 配置后仍然报错？

**A:** 检查以下几点：
1. API Key 是否正确复制（没有多余空格）
2. API Key 是否有效（未过期）
3. 网络连接是否正常
4. 是否选择了正确的 provider

### Q3: 如何批量处理多个文件？

**A:** 可以编写简单的 Shell 脚本：

```bash
#!/bin/bash
for file in 00-每日工作区/01-待处理材料/*.md; do
    python main.py analyze --file "$file" --archive
done
```

### Q4: 知识库索引损坏怎么办？

**A:** 使用 Python 交互式重建索引：

```python
from workflow_automation import KnowledgeManager

km = KnowledgeManager()
km.rebuild_index()
```

### Q5: 如何自定义输出模板？

**A:** 修改 `workflow_automation/task_scheduler.py` 中的模板生成函数，或者在 `templates/` 目录下创建自定义模板。

## 最佳实践

### 1. 每日工作流程

**早上（9:00）：**
```bash
# 生成今日任务
python main.py daily

# 打开任务文件，填写今日重点
```

**上午（9:30-11:30）：**
- 按 PIDST 框架采集信息
- 将材料暂存到 `01-待处理材料/`

**中午（11:30-12:00）：**
```bash
# 批量分析材料
for file in 00-每日工作区/01-待处理材料/*.md; do
    python main.py analyze --file "$file" --archive
done
```

**下午（14:00-16:00）：**
```bash
# 提炼洞察
python main.py insight \
  --file "00-每日工作区/03-三行摘要收集.md" \
  --min-insights 3 \
  --archive

# 构建证据链（如有专题研究）
python main.py evidence \
  --topic "研究主题" \
  --file "相关信息.md" \
  --archive
```

**晚上（17:30-18:00）：**
```bash
# 生成复盘
python main.py review

# 查看统计
python main.py stats
```

### 2. 信息质量控制

**采集阶段：**
- 优先使用官方和权威来源
- 明确标注信息时间和来源
- 避免传播未经验证的信息

**分析阶段：**
- 每条信息都要完成三行摘要
- 至少提炼 3 个核心洞察
- 构建完整的证据链（F-D-P-T-S）

**沉淀阶段：**
- 及时归档到知识库
- 添加准确的标签
- 建立知识关联

### 3. 知识库维护

**每日：**
- 及时归档新增内容
- 更新知识库索引

**每周：**
```bash
# 生成周总结
python main.py weekly --start-date YYYY-MM-DD --end-date YYYY-MM-DD

# 检查知识库统计
python main.py stats
```

**每月：**
- 清理过时内容
- 优化标签体系
- 重建知识库索引

### 4. AI 使用技巧

**提高分析质量：**
- 提供完整的上下文信息
- 明确指定分析维度
- 使用 `--min-insights` 参数控制洞察数量

**优化性能：**
- 使用本地模型处理简单任务
- 批量处理相似任务
- 合理设置 `temperature` 和 `max_tokens`

**成本控制：**
- 优先使用本地模型
- 避免重复分析相同内容
- 使用更小的模型处理简单任务
