# 提示词库去重合并

## 目标

清理 `05-AI技术文档/提示词收集库/` 与 `05-AI技术文档/提示词工程/` 之间的重复文件，
保留更完整的版本，最终将所有提示词统一归入 `提示词工程/` 目录。

## 执行流程

### Step 1: 扫描两个目录

分别用 Glob 搜索：
- `05-AI技术文档/提示词收集库/**/*.md`
- `05-AI技术文档/提示词工程/**/*.md`

列出两个目录下的所有文件。

### Step 2: 识别重复

对比规则（按优先级）：
1. **同名文件**：文件名完全相同（忽略路径）
2. **近似文件名**：去除后缀/前缀后名称相同（如 `哈雷酱.md` vs `哈雷酱-v2.md`）
3. **内容相似**：读取文件内容，如果前 200 字符相似度 >80%，视为重复

### Step 3: 展示去重计划

以表格形式展示每对重复文件的对比结果，等待用户确认：

```
📋 提示词去重计划：

| 文件A (收集库) | 文件B (工程) | 字数对比 | 建议 |
|---------------|-------------|---------|------|
| 哈雷酱.md (1200字) | 哈雷酱.md (2500字) | A < B | 保留B |
| 真理之猫.md (800字) | - | 独有 | 迁移到工程/ |
| ... | ... | ... | ... |

(哼，确认后本小姐才会动手！)
```

### Step 4: 执行合并

用户确认后：
1. **重复文件**：保留字数更多/内容更完整的版本，删除另一个
2. **收集库独有文件**：移动到 `提示词工程/` 对应子目录
3. **工程独有文件**：保持不动

子目录归类规则：
```
含 "角色/人格/persona" → 提示词工程/角色与人格/
含 "分析/研究/方法" → 提示词工程/分析与研究/
含 "创意/写作/绘画" → 提示词工程/创意与写作/
其他 → 提示词工程/通用/
```

### Step 5: 清理空目录

如果 `提示词收集库/` 已清空，保留目录但创建一个 `_DEPRECATED.md` 说明文件：
```markdown
> 本目录已废弃，所有提示词已迁移至 [[05-AI技术文档/提示词工程/]]
```

### Step 6: 报告

输出合并结果：
- 发现重复对数
- 保留文件数
- 迁移文件数
- 删除文件数

## 安全机制

- 合并前必须展示完整计划并获得用户确认
- 删除操作逐个确认
- 对于内容差异较大的"疑似重复"，标记为需人工审核而非自动处理
