# 材料沉淀为知识条目

## 目标

扫描 `00-每日工作区/01-待处理材料/` 中的原始材料，将其转化为结构化知识条目并归档到对应模块。

## 执行流程

### Step 1: 扫描待处理材料

用 Glob 搜索 `00-每日工作区/01-待处理材料/**/*.md`，列出所有待处理文件。

### Step 2: 逐个分析

对每个文件：
1. 读取内容，判断主题领域
2. 确定目标模块（02-政策/03-行业/04-专题/06-个人成长）
3. 提取核心信息，填充知识条目模板

### Step 3: 展示转化计划

以表格形式展示每个文件的转化方案：

```
📋 材料沉淀计划：

| 原始文件 | 主题 | → 目标模块 | 操作 |
|---------|------|-----------|------|
| xxx.md | 政策分析 | 02-政策于宏观库/ | 转化 |
| yyy.md | 内容不足 | - | 跳过 |

(哼，确认后本小姐才会动手！)
```

### Step 4: 执行转化

用户确认后，对每个文件：
1. 读取 `templates/obsidian/知识条目模板.md` 获取标准格式
2. 基于原始内容填充模板字段：
   - 一句话结论（从内容中提炼）
   - 关键证据（提取事实性内容）
   - 标签（根据目标模块自动分配）
3. 将生成的知识条目写入目标模块目录
4. 在原始文件开头标注 `> ✅ 已沉淀至 [[目标文件]]`

### Step 5: 报告

输出转化结果：
- 处理文件数 / 跳过文件数
- 按目标模块分组统计

## 安全机制

- 不删除原始材料，仅标注已处理
- 转化前必须获得用户确认
- 内容不足 50 字的文件自动跳过
