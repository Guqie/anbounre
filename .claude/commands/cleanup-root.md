# 根目录清理归档

## 目标

将根目录中散落的非系统文件归档到知识库对应模块目录。

## 系统文件白名单（不动）

以下文件属于系统配置，保留在根目录：
- `README.md`
- `CLAUDE.md`
- `_MOC-知识库总览.md`

## 归档规则

按以下映射执行文件移动：

```
HarleyPersona.md                        → 09-灵魂系统/哈雷记忆/
观察思索与小城青年社交结构的转移.md       → 06-个人成长/
工作流自动化系统使用指南.md              → 07-完成归档/
PROJECT_KNOWLEDGE.md                     → 07-完成归档/
说明文档.md                              → 07-完成归档/
```

## 执行流程

### Step 1: 扫描根目录

用 Glob 搜索根目录下的 `*.md` 文件（非递归），排除白名单文件。
同时扫描 `*.canvas`、`*.base` 等非 md 散落文件。

### Step 2: 展示归档计划

列出每个文件的当前位置和目标位置，等待用户确认。

格式：
```
📦 根目录清理计划：

| 文件 | → 目标位置 | 操作 |
|------|-----------|------|
| HarleyPersona.md | 09-灵魂系统/哈雷记忆/ | 移动 |
| 说明文档.md | 07-完成归档/ | 移动 |
| 未命名.canvas | - | 删除（确认后） |
| ... | ... | ... |

(哼，确认后本小姐才会动手！)
```

### Step 3: 执行归档

用户确认后：
1. 确保目标目录存在（不存在则创建）
2. 使用 `git mv` 移动已跟踪文件（保留 git 历史）
3. 使用普通 `mv` 移动未跟踪文件
4. 对于标记删除的文件，二次确认后删除

### Step 4: 更新双链引用

移动文件后，搜索知识库中所有引用了被移动文件的 `[[双链]]`：
- 如果使用的是文件名（无路径），Obsidian 会自动解析，无需修改
- 如果使用了完整路径引用，需要更新路径

### Step 5: 报告

输出归档结果：
- 移动了多少文件
- 删除了多少文件
- 根目录剩余文件列表

## 安全机制

- 移动前必须展示完整计划并获得用户确认
- 删除操作需要二次确认
- 不触碰 `.git/`、`.obsidian/`、`.claude/` 等隐藏目录
