# 孤立文档双链修复

## 目标

扫描知识库中无入站双链的孤立文档，为其建立至少一条双链关系。

## 排除范围

以下文件不视为"孤立"：
- `_MOC-*` 索引页（本身就是链接枢纽）
- `README.md`、`CLAUDE.md`（系统文件）
- `templates/` 下的模板文件

## 执行流程

### Step 1: 识别孤立文档

扫描所有 `.md` 文件，统计每个文件的入站双链数（被其他文件 `[[引用]]` 的次数）。
入站链接为 0 的文件即为孤立文档。

### Step 2: 分类孤立文档

按所属模块分组展示：

```
🏝️ 孤立文档清单（共 X 个）：

📁 00-每日工作区 (X个)
  - 文件A.md
  - 文件B.md

📁 05-AI技术文档 (X个)
  - 文件C.md
  ...
```

### Step 3: 自动链接策略

对每个孤立文档，按优先级尝试建立链接：

1. **MOC 收录**：将文档链接添加到其所属模块的 `_MOC-*.md` 索引页
2. **关联文档互链**：读取文档内容，在同模块中寻找主题相关的文档，互相添加双链
3. **返回链接**：在文档末尾添加 `**返回** [[_MOC-知识库总览]]`

### Step 4: 展示链接计划

```
🔗 双链修复计划：

| 孤立文档 | 链接方式 | 目标文档 |
|---------|---------|---------|
| 文件A.md | MOC收录 | _MOC-大脑核心 |
| 文件B.md | 互链 | 文件X.md |
| 文件C.md | 返回链接 | _MOC-知识库总览 |

(哼，确认后本小姐才会动手！)
```

### Step 5: 执行链接

用户确认后批量执行编辑操作。

### Step 6: 报告

输出修复结果：
- 处理孤立文档数
- MOC 收录数 / 互链数 / 返回链接数
- 剩余孤立文档数

## 安全机制

- 每次编辑前展示计划并获得确认
- 不修改 08/09 模块的文档结构
- 仅追加链接，不删除或修改现有内容
