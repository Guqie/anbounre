## 文档类型识别规则

### 技术文档
- 关键词: 技术、工具、教程、指南、配置、API
- 目标: `05-AI技术文档/技术文档/`
- 标签: `#类型/工具技巧` `#领域/技术`

### 研究报告
- 关键词: 行业、政策、分析、报告、研究
- 目标: `03-行业研究库/` 或 `04-专题研究库/`
- 标签: `#类型/研究` `#领域/行业` 或 `#领域/政策`

### 方法论
- 关键词: 方法、框架、模型、原则
- 目标: `01-研究方法论/`
- 标签: `#类型/方法论`

### 提示词
- 关键词: 提示词、prompt、system prompt
- 目标: `05-AI技术文档/提示词工程/`
- 标签: `#类型/提示词`

### 记忆文档
- 关键词: 记忆、画像、偏好、决策
- 目标: `08-大脑核心/`
- 标签: `#类型/记忆`

### 日记
- 关键词: 日记、感想、思考
- 目标: `10-个人日记/`
- 标签: `#类型/日记`

## Frontmatter 标准

```yaml
---
tags:
  - 类型/[工具技巧|研究|方法论|提示词|记忆|日记]
  - 状态/[活跃|归档|草稿]
  - 领域/[技术|政策|行业|宏观]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

## MOC 文件规范

- 命名: `_MOC-[模块名].md`
- 位置: 各模块根目录
- 内容: 模块文档索引、快速导航、核心洞察

## 双链规范

- 使用 `[[文档标题]]` 格式
- 避免使用完整路径
- 优先链接到 MOC 文件
