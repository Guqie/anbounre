## 一、核心设计理念（借鉴claude-mem）

### 1.1 渐进式披露（Progressive Disclosure）
**原则：** 不要一次性加载所有记忆，按需获取

**实施方案：**
- 第1层：记忆索引（标题+ID+时间）
- 第2层：记忆摘要（关键信息）
- 第3层：完整内容（详细记录）

**优势：**
- 减少token消耗
- 提高检索效率
- 更好的用户体验

---

### 1.2 自动化记忆捕获
**原则：** 无需手动干预，自动记录重要信息

**实施方案：**
- 使用生命周期钩子自动捕获
- 在关键时刻记录状态
- 自动生成摘要

**钩子设计：**
1. **SessionStart** - 会话开始时加载相关记忆
2. **UserPromptSubmit** - 用户提示后记录上下文
3. **PostToolUse** - 工具使用后记录结果
4. **SessionEnd** - 会话结束时保存记忆

---

### 1.3 混合搜索架构
**原则：** 结合全文搜索和语义搜索

**实施方案：**
- SQLite FTS5 - 全文搜索
- 向量数据库 - 语义搜索
- 混合排序算法

## 二、技术架构设计

### 2.1 数据存储

**主数据库：SQLite**
```sql
-- 会话表
CREATE TABLE sessions (
    id INTEGER PRIMARY KEY,
    start_time TIMESTAMP,
    end_time TIMESTAMP,
    summary TEXT
);

-- 观察表（记忆条目）
CREATE TABLE observations (
    id INTEGER PRIMARY KEY,
    session_id INTEGER,
    timestamp TIMESTAMP,
    type TEXT,  -- conversation, tool_use, decision, insight
    title TEXT,
    content TEXT,
    summary TEXT,
    tags TEXT,
    priority INTEGER,  -- P0-P3
    FOREIGN KEY (session_id) REFERENCES sessions(id)
);

-- 全文搜索索引
CREATE VIRTUAL TABLE observations_fts USING fts5(
    title, content, summary, tags
);
```

**向量数据库：** 考虑使用Chroma或简化版本

---

### 2.2 记忆分层结构

**P0 - 核心记忆（永久）**
- 用户基本信息
- 重要关系
- 核心价值观
- 存储：MEMORY.md + SQLite

**P1 - 重要记忆（长期）**
- 工作项目
- 重要决策
- 学习成果
- 存储：memory/主题.md + SQLite

**P2 - 一般记忆（中期）**
- 日常对话
- 临时任务
- 存储：memory/YYYY-MM-DD.md + SQLite

**P3 - 临时记忆（短期）**
- 会话内容
- 临时计算
- 存储：仅SQLite，定期清理

## 四、与claude-mem的对比

### 4.1 相似之处
- ✅ 渐进式披露理念

- ✅ 自动化记忆捕获
- ✅ 混合搜索架构

### 4.2 差异之处
- 📝 我们使用Markdown文件 + SQLite
- 📝 claude-mem使用纯数据库
- 📝 我们更注重人类可读性
- 📝 claude-mem更注重自动化

### 4.3 我们的优势
- 记忆文件人类可读
- 易于备份和版本控制
- 可以手动编辑
- 与现有工作流兼容
