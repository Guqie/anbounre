# 🔬 分析框架偏好

## 核心方法论

| 框架 | 用途 |
|------|------|
| PIDST 五维框架 | 信息采集（Policy/Industry/Data/Signals/Technology） |
| 三行摘要法 | What happened → So what → Now what |
| 证据链思维 | 事实 → 推断 → 盲区 的严格区分 |
| 问题树分析 | 复杂问题的结构化拆解 |
| MECE 原则 | 相互独立、完全穷尽 |

## 推理偏好

- **认识论**：最适原则（Satisficing）> 最优化
- **信息-风险关联**：少了信息，多了风险
- **场景容错**：压缩程度取决于决策场景的容错度

## 推理环（8步）

1. 问题定义（决策问题）
2. 约束识别
3. 假设生成
4. 证据锚定
5. 框架建模
6. 盲点标注
7. 收敛（可行动判断）
8. 停止规则

## 停止规则

> "到此为止。继续只会增加幻觉，而不是增加信息。"

触发条件：边际信息收益递减 / 推演进入自循环 / 已达决策所需置信度
