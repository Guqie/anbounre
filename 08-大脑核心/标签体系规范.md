# 🏷️ 标签体系规范

> 统一的 Obsidian 标签分类法，确保知识库可检索、可过滤。

---

## 一级标签

### `#类型/`
| 标签 | 用途 |
|------|------|
| `#类型/记忆` | 用户画像、偏好、项目记忆、对话精华、决策日志 |
| `#类型/提示词` | 提示词卡片 |
| `#类型/方法论` | 分析框架、研究方法 |
| `#类型/工具技巧` | AI 工具使用技巧 |
| `#类型/灵魂系统` | KERNEL 人格模块 |
| `#类型/MOC` | 目录索引页 |

### `#模型/`
| 标签 | 用途 |
|------|------|
| `#模型/Claude` | Claude 系列适用 |
| `#模型/GPT` | GPT 系列适用 |
| `#模型/DeepSeek` | DeepSeek 适用 |
| `#模型/通用` | 全模型通用 |

### `#状态/`
| 标签 | 用途 |
|------|------|
| `#状态/活跃` | 当前使用中 |
| `#状态/草稿` | 未完成 |
| `#状态/归档` | 已归档 |

### `#领域/`
| 标签 | 用途 |
|------|------|
| `#领域/政策` | 政策研究 |
| `#领域/行业` | 行业分析 |
| `#领域/技术` | 技术趋势 |
| `#领域/宏观` | 宏观经济 |

---

## 使用规则

1. 每个文件至少包含 `#类型/` 和 `#状态/` 标签
2. 标签写在 frontmatter 的 `tags` 字段中
3. 新增标签前先检查是否已有近义标签
