# 决策：代码仓库 × 知识库仓库完全分离

> 日期：2026-02-23
> 状态：✅ 已执行
> 决策者：用户 + 哈雷酱

---

## 背景

`无敌战警1.0/` 根目录下代码（Python CLI + Flask RSS 网关 + Agent 系统）和 Obsidian 知识库（00-09 编号目录）混杂，导致：
- Obsidian 图谱被 `.py`、`node_modules` 等代码文件污染
- git 历史中代码变更与知识库变更混杂，难以追溯
- 代码通过相对路径直接读写知识库目录，耦合过紧

## 决策

将项目拆分为两个独立仓库：

| 仓库 | 路径 | 内容 |
|------|------|------|
| 代码仓库 | `D:/桌面/无敌战警-engine/` | Python CLI, Flask, Agents, 配置 |
| 知识库 | `D:/桌面/无敌战警1.0/` | Obsidian Vault, 00-09 目录 |

## 关键变更

1. `config.yaml` 新增 `vault_root` 字段指向知识库绝对路径
2. `utils.py` 的 `load_config()` 自动拼接 `vault_root` + 相对目录
3. 知识库中所有代码文件已删除
4. 两个仓库各自独立 git 管理

## 影响

- 代码变更不再污染知识库 git 历史
- Obsidian 图谱视图干净
- 部署时需确保 `vault_root` 路径正确
