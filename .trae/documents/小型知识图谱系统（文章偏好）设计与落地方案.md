### 二、功能概述与验收标准（前后端完整）

#### 前端（Vite+React+CSS）

* Intake（采集）：字段填写与校验（URL/日期），domain chips 快选与自定义；草稿自动保存；清空/加载示例。

  * 验收：校验通过后显示“流式分析”可点；刷新后草稿恢复成功。

* Analyze（流式分析）：SSE 展示思考与回复（自动滚动/暂停/清空）；解析 `answer` 为 JSON；校验并高亮错误。

  * 验收：持续显示事件；JSON 合法自动填充编辑器；非法显示错误路径与消息。

* Import（入库）：编辑 JSON；校验通过后入库；显示结果或错误详情（约束冲突/连接失败）。

  * 验收：成功返回 `ok:true` 并在 Neo4j 可见；错误明确指向冲突字段。

* 交互优化：快捷键（Ctrl+Enter/ESC/Ctrl+S）；响应式布局与可访问性；状态样式（loading/error/success）。

#### 后端（FastAPI）

* `/api/analyze/stream`：OpenAI 客户端（DashScope 兼容）调用 `deepseek-v3.2-exp`，开启思考模式与流式；逐行输出 `reasoning/answer/usage`。

  * 验收：SSE 输出稳定；网络中断自动重试；失败可回退 `/api/analyze`。

* `/api/analyze`：一次性返回；解析为 JSON 并基础校验。

  * 验收：合法返回 200；不合法返回 400 与原文。

* `/api/import`：参数化 Cypher 幂等写入；返回结果与错误详情。

  * 验收：重复入库不生成重复节点；错误带 `error/detail` 字段。

### 三、前端详设（UI/交互/状态机/优化）

* 页面布局（线框）：Intake（左右两列）、Analyze（左流式右答与 JSON）、Import（左 JSON 预览右入库控制）。

* 组件职责与状态机：

  * IntakeForm：`idle → validating → ready`；校验/chips/草稿（节流 500ms）。

  * StreamViewer：`connecting → streaming → paused → done → failed`；分片渲染与自动滚动；暂停/清空；失败回退。

  * JsonEditor：`ajv` 校验；错误路径高亮；快捷修复空数组/缺失字段；草稿保存。

  * ImportButton：`disabled → importing → success|error`；结果提示与错误详情面板。

* CSS 策略：主题变量（`--bg/--fg/--primary/--muted/--error/--success`）；Grid/Flex；高对比度与焦点可见。

* 性能预算：首屏 <150KB；SSE 分片渲染；JSON 校验节流/idle；localStorage 草稿保底。

### 四、后端详设（API 合约/流式协议/安全）

* 合约：

  * `POST /api/analyze/stream`：请求 Intake JSON；响应 `text/event-stream` 行式 JSON（`reasoning/answer/usage`）。

  * `POST /api/analyze`：一次性返回合法 JSON 或 400。

  * `POST /api/import`：`user_id/user_name/payload`；返回 `ok` 或 `error+detail`。

* 流式协议：逐行 JSON；前端行级解析；中断自动重试；失败回退非流式。

* 安全：API Key 仅后端环境变量；前端经代理 `/api`；生产使用 HTTPS 与最小权限的 Neo4j 账户。

### 五、图谱建模、约束与 JSON Schema

* 节点/关系：`Article/Domain/Topic/Institution/ActorType/Geo/Keyword/User`；评分关系 `:RATED` 含维度分与时间戳。

* 唯一约束与索引：`article_id` 与各 `name` 字段；脚本集中于 `neo4j/constraints.cypher`。

* JSON Schema（前后端一致）：严格字段与类型规则；入库前必须校验通过。
