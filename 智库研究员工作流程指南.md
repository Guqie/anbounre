# 智库研究员工作流程指南

## 概述

本指南基于"六步日常循环"工作法，为智库研究员提供结构化的每日工作流程，涵盖信息采集、分析处理、观点提炼和知识沉淀的完整链条。

## 核心工作流程

### 第一步：任务定义（3分钟）

**目标：** 明确当日核心任务，避免盲目忙碌

**操作步骤：**
1. 打开 `00-每日工作区/00-今日任务.md`
2. 列出今日最重要的1-3个核心任务
3. 按任务类型分类：
   - 政府库/行业库更新
   - 决策支持研究
   - 专题研究推进
4. 明确今日必须交付的最小输出物（MVP）

**模板：**
```markdown
# 今日任务 - [日期]

## 今日最重要的三件事（MIT）
1. [具体任务1]
2. [具体任务2] 
3. [具体任务3]

## 任务类型分类
- ### 1. 基础库维护（政府库/行业库）
- - [ ] 政策采集：[目标数量]条
- - [ ] 行业动态：[目标数量]条
- - [ ] 数据库上传进度：[具体动作]
- 
- ### 2. 信顾服务（信息追踪）
- - [ ] 重点监测对象：[企业/政策/话题]
- - [ ] 交付物：[日报/周报/快讯]
- 
- ### 3. 深度研究（研报撰写）
- - [ ] 进行中项目：[项目名称]
- - [ ] 今日进度：[章节/图表/数据分析]

## 今日最小交付物
- [明确的输出物描述]
```

### 第二步：结构化信息采集（45-90分钟）

**目标：** 按PIDST五维框架系统性收集信息

#### 2.1 PIDST五维检索

**P (Policy) - 政策层面**
- 部委级政策：国家发改委、工信部、卫健委等
- 省级政策：重点省市政策动态
- 市级政策：地方实施细则

**I (Industry) - 产业层面**
- 医疗健康产业链动态
- 人才教育产业发展
- 相关产业竞争格局

**D (Data) - 数据层面**
- 官方统计数据
- 行业报告数据
- 第三方调研数据

**S (Signals) - 信号层面**
- 舆情监测
- 弱信号识别
- 企业动向

**T (Technology) - 技术层面**
- AI+医疗技术进展
- AI+教育应用创新
- 相关技术趋势

#### 2.2 信息处理流程

1. **信息暂存**
   - 将收集到的信息先存入 `00-每日工作区/01-待处理材料.md`
   - 按PIDST分类整理

2. **三行摘要处理**
   - 对每条信息进行三行摘要
   - 存入 `00-每日工作区/03-三行摘要收集.md`

**三行摘要标准格式：**
```markdown
### [材料名称]
**1. What happened：** [发生了什么]
**2. So what：** [意味着什么]  
**3. Now what：** [下一步如何]
**标签：** 政策/行业/技术/数据
**时间：** [信息时间]
**来源：** [信息来源]
```

### 第三步：同步初步分析（30分钟）

**目标：** 对收集的信息进行初步分析和归类

**操作步骤：**
1. **信息归类**
   - 按重要程度分级：🔴高 🟡中 🟢低
   - 按分析维度分类：供给/需求/市场/竞争/政策

2. **关联分析**
   - 识别信息间的关联性
   - 发现潜在的趋势和模式
   - 标记需要深入研究的问题

3. **弱信号识别**
   - 识别可能的新兴趋势
   - 记录异常或反常现象
   - 评估信号的可信度和重要性

### 第四步：观点提炼（15分钟）

**目标：** 基于信息分析提炼核心观点和洞察

**操作方法：**
1. **结论前置**
   - 每日至少提炼3个核心洞察
   - 每个洞察都要有充分的证据支撑

2. **洞察模板：**
```markdown
### 洞察 [序号]：[核心观点]
**依据：** [支撑证据]
**逻辑链：** [推理过程]
**趋势：** [未来走向]
**影响：** [可能影响]
```

### 第五步：结构化表达（30-60分钟）

**目标：** 根据任务类型产出相应的结构化内容

#### 5.1 基础库维护（政府库/行业库）

**目标：** 确保双库信息的全面性、及时性和准确性

**核心动作：**
1. **政策采集（政府库）**
   - 重点扫描：发改委、工信部、卫健委等部委官网
   - 采集标准：正式文件 > 征求意见稿 > 政策解读
   - 必填字段：发布机构、发布日期、文号、原文链接

2. **行业动态（行业库）**
   - 重点扫描：行业协会、头部企业官网、权威研报
   - 采集标准：重大事件 > 数据披露 > 人事变动
   - 必填字段：涉及主体、事件类型、关键数据

**输出物：** 数据库上传记录 / 每日更新摘要

**模板：**
```markdown
# [库名]每日更新摘要 - [日期]

## 📊 更新统计
- 新增条目：[数量]条
- 重点覆盖领域：[领域1]、[领域2]

## 🚨 重点内容速览
1. **[政策/事件名]**
   - 核心内容：[简要说明]
   - 价值等级：⭐⭐⭐
   - 标签：[标签1] [标签2]
```

#### 5.2 信顾服务（信息追踪）

**目标：** 为客户提供高价值的信息追踪与预警服务

**核心动作：**
1. **监测对象追踪**
   - 针对特定企业/话题的全网监测
   - 识别负面舆情与潜在风险

2. **交付物制作**
   - 撰写日报/周报/快讯
   - 提供“智库点评”增值服务

**输出物：** 信顾快报 / 监测日报

**模板：**
```markdown
# [客户/领域] 每日信顾快报 - [日期]

## 🚨 紧急/重要预警
- [需立即关注的风险或机会]

## 📰 核心动态
1. [动态1]：[简要解读]
2. [动态2]：[简要解读]

## 💡 智库点评
- [针对今日动态的专业建议]
```

#### 5.3 深度研究（研报撰写）

**目标：** 推进深度研究报告的撰写进度

**核心动作：**
1. **模块化推进**
   - 每日聚焦一个章节或图表
   - 数据清洗与可视化制作

2. **逻辑自洽检查**
   - 检查证据链是否完整
   - 检查观点是否鲜明

**输出物：** 研报章节草稿 / 数据图表

### 第六步：知识库沉淀与复盘（20分钟）

**目标：** 将当日工作成果沉淀到知识库，并进行复盘

#### 6.1 知识库更新

**操作清单：**
- □ 今日三行摘要 → 分类存入各专业库
- □ 今日弱信号 → 更新弱信号库
- □ 今日观点 → 更新观点库（Insights）
- □ 今日模型 → 更新方法论库
- □ 今日专题内容 → 更新专题文件夹

#### 6.2 日终复盘

**复盘模板：**
```markdown
# 今日复盘 - [日期]

## 今天做得好的3件事
1. [具体成果1]
2. [具体成果2]
3. [具体成果3]

## 今天遇到的问题
- [问题1及解决方案]
- [问题2及解决方案]

## 明日优化点
- [改进方向1]
- [改进方向2]

## 明日重点任务
1. [明日任务1]
2. [明日任务2]
3. [明日任务3]
```

## 专项工作流程

### 企业跟踪分析（如中远海运）

**问题树构建：**
```
主问题：[企业名称]近期趋势？
├── 1. 宏观环境与行业景气度
├── 2. 公司经营状况
│   ├── 订单情况
│   ├── 航线布局
│   ├── 财报表现
│   └── 投资动向
├── 3. 风险信号
│   ├── 地缘政治风险
│   ├── 港口运营风险
│   └── 供应链风险
└── 4. 政策导向影响
```

**证据链收集：**
- 事实（F）：客观发生的事件
- 数据（D）：量化指标和统计数据
- 政策（P）：相关政策文件和导向
- 趋势（T）：行业和市场趋势
- 弱信号（S）：早期预警信号

- 用本地模型做自动摘要
- 用Python写信息清洗脚本
- 用AI对比政策条款
- 用Agent辅助构建问题树

## 质量控制标准

### 信息质量标准
1. **来源可靠性**：优先使用官方和权威来源
2. **时效性**：信息应为最新或明确标注时间
3. **完整性**：信息应包含关键要素
4. **准确性**：避免传播未经验证的信息

### 分析质量标准
1. **逻辑性**：分析推理过程清晰
2. **证据性**：观点有充分证据支撑
3. **客观性**：避免主观臆断
4. **前瞻性**：具有预测和指导价值

### 输出质量标准
1. **结构化**：内容组织清晰有序
2. **可操作**：建议具体可执行
3. **简洁性**：表达简明扼要
4. **专业性**：符合智库研究标准

## 常见问题与解决方案

### 信息过载问题
**症状：** 收集了大量信息但无法有效处理
**解决方案：**
- 严格按PIDST框架筛选信息
- 设定每日信息处理上限
- 优先处理高价值信息

### 分析深度不足
**症状：** 停留在信息汇总层面，缺乏深度分析
**解决方案：**
- 强制执行三行摘要法
- 每日必须产出至少3个洞察
- 建立证据链思维

### 知识沉淀不足
**症状：** 每日工作无法形成知识积累
**解决方案：**
- 严格执行知识库更新流程
- 定期回顾和整理知识库
- 建立知识关联和索引
