## 1. 系统定位

**无敌战警系统**是一个面向智库研究员的智能知识管理与工作流自动化系统，由两个核心仓库组成：

| 仓库 | 职责 | 技术栈 |
|------|------|--------|
| **无敌战警1.0**（知识库层） | 结构化知识存储、双链网络、记忆与人格系统 | Obsidian, Markdown |
| **无敌战警-engine**（代码层） | 上下文编排、工作流自动化、AI 调用 | Python, Context OS v3.0 |

---

## 2. 核心价值

- **AI 外部大脑**：让 AI 跨会话记住用户画像、偏好、上下文
- **智能上下文编排**：Context OS v3.0 实现 Intent-Driven、Budget-Aware 的上下文注入
- **工作流自动化**：从信息采集到知识沉淀的全流程自动化
- **知识沉淀**：六步日常循环，持续积累可复用的知识资产

## 3. Subagent 网络架构（v2.0）

核心架构：**一个以三核为本体、以 Copilot 为主代理、以 Subagent 为执行网络、以 Gateway/Scheduler 为调度中枢的认知系统。**

### 三核底座（Kernels）

| 核心 | 职责 | 知识库位置 |
|------|------|-----------|
| **Persona Kernel（人格核）** | 角色底色、表达风格、职业身份、技能包、方法包、输出协议 | `09-灵魂系统/` |
| **Memory Kernel（记忆核）** | 记忆分层、访问权限、召回策略、写回治理 | `08-大脑核心/` |
| **Evolution Kernel（演进核）** | 反思机制、演进规则、能力升级审批 | 演进日志与规则配置 |

### 代理层（Agent Network）

| 代理 | 职责 | 权限等级 |
|------|------|----------|
| **Copilot** | 总协调代理，任务理解、Subagent 选择、上下文编排、结果汇总 | 高 |
| **Memory Agent** | 记忆召回、压缩、写回、治理 | 高（可写回用户记忆） |
| **Research Agent** | 信息搜集、多源比对、洞察提炼 | 中（只读知识库） |
| **Decision Agent** | 方案比较、风险识别、判断形成 | 高（可写决策记忆） |
| **Writing Agent** | 结构化成文、风格适配、表达优化 | 低（只写输出文档） |
| **Reflection Agent** | 任务复盘、经验提炼、演进线索 | 中（写反思记录） |
| **Evolution Agent** | 能力升级、规则优化、系统演进 | 最高（可修改核心配置） |

### 调度层（Gateway / Scheduler）

| 模块 | 职责 | 关键能力 |
|------|------|----------|
| **Entry Gateway** | 统一入口网关 | 事件监听、命令解析、请求验证 |
| **Session Manager** | 会话生命周期管理（8 阶段） | 会话创建、状态维护、生命周期管理 |
| **Session Router** | 任务路由（7 种任务模式） | 任务分类、模式选择、路由决策 |
| **Agent Dispatcher** | 代理分发（5 种协作模式） | Subagent 选择、协作编排、上下文构建 |
| **Channel Adapter** | 渠道适配 | 多渠道协议（Obsidian/CLI/WebSocket/REST） |
| **Plugin/Tool Runtime** | 工具运行时 | 工具注册、调用、结果缓存 |
| **Budget Controller** | 预算治理 | 全局 Budget 控制、5 层降级策略 |
| **Observability & Release** | 可观测性与灰度发布 | 日志、指标、追踪、Feature Flag、灰度发布 |

详见：[Gateway/Scheduler 详细设计](../design/02-gateway-scheduler.md)
